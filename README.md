# 대규모 NPC 길찾기 시스템 개발 계획서


## 주차별 개발 일정

| 주차 | 주제 | 세부 목표 |
|------|------|-----------|
| 1주차 | 엔진 준비 | MiniEngine 구조 분석 / 맵 모델 로드 | 단일 맵 + 단일 캐릭터(캡슐)가 화면에 렌더링되는 빌드 |
| 2주차 | 복셀화 및 인스턴싱 | 로드된 맵 메시를 복셀 그리드로 변환 / GPU 인스턴싱 기반 캐릭터 렌더링 파이프라인 구축 / 1,000개 동시 로드를 위한 최적화 | 1,000개 NPC 동시 렌더링 데모 |
| 3주차 | Flow Field 코어 + 그룹 선택 | 복셀 기반 Flow Field 알고리즘 구현 / 드래그를 통한 NPC 그룹 선택(박스 셀렉션) / 클릭을 통한 목적지 지정 및 이동 명령 | 그룹 선택 + 단순 이동 데모 |
| 4주차 | 다수 NPC 길찾기 | Flow Field를 이용한 다수 NPC 동시 길찾기 구현 |
| 5주차 | 병목 지형 처리 | 좁은 통로, 병목 구간에서의 NPC 길찾기 안정화 / 정체 및 교착 상태 해소 로직 | 병목 지형 통과 안정성 테스트 |
| 6주차 | 동적 지형 대응 | 런타임 지형 변화(장애물 추가/제거) 구현 / 변화에 따른 복셀맵 다이나믹 캐싱 및 Flow Field 재계산 최적화 | 동적 장애물 회피 데모 |
| 7주차 | 그룹 간 상호작용 | 서로 다른 NPC 그룹이 교차/통과하는 패턴 구현 | 다중 그룹 교차 이동 데모 |
| 8주차 | 테스트 및 안정화 | 최종 빌드 |
